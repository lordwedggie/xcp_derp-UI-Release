## xcpDerpNodes — Hard-Learned Lessons

### 1. Diagnosis before coding
Map the pipeline end-to-end with logs BEFORE changing anything. One console.log at each stage pinpoints the blocker in one turn. Never guess.

### 2. Surgical edits — one file at a time
If a fix touches 3+ files, pause. Spraying changes creates self-inflicted bugs that cost more turns to debug than the original problem.

### 3. Don't break code while inserting diagnostics
Adding console.warn lines must preserve original logic exactly. In this case: replaced layout.compute() with a log, silently disabling removal-path force-height.

### 4. No ES module exports in ComfyUI custom node scripts
ComfyUI/Vite loads these as pooled scripts; `export const`/`export function` cause "Unexpected token 'export'" and "Identifier already declared". Use `window.fn = function(...){}` for shared utilities. Use `var` (not `const`) for module-level constants.

### 5. Verify after every reset
After git reset --hard, run `git diff` against the target branch to confirm ALL changes are present. Glance at each file's key sections — don't assume.

### 6. Fix at the source, not around it
derpLoraStack was not the problem. `resolveRuntimeDockSize` in dockDimensions.js was freezing height for horizontal docks via `Math.max(storedH, liveH, 0)`. Fix: `autoHeight ? engineFloorH : Math.max(storedH, liveH, 0)`. Framework fixes at the source are robust; workarounds at the symptom are fragile.

### 7. Avoid change amplification
When a simple fix doesn't work, resist adding MORE changes. Instead: revert all, verify baseline, add ONE change, test, repeat.

## Drag-and-Drop Ghost Height Pattern (derpSignalOut lesson)

**Problem:** Ghost row height must match original. `height: "auto"` recomputes from children; hardcoded values won't match.

**Pattern (3-step):**
1. Record original region height at dragStart into snapshot (e.g. `_signalOutFloatingSnapshot = { baseReg: { h: region.h } }`)
2. Ghost uses snapshot: `height: item.isPreviewGhost ? dragPlaceholderHeight : "auto"`
3. Clear snapshot on dragEnd/press

**Do NOT:** hardcode ghost height, use `height: "auto"` for ghost, or use `alpha:0` with frozen layout

**Symptoms of wrong height:** REGION border as 1-2px line, pulsing animations, children shrunk

**Debugging:** temporarily make ghost visible (alpha 0.5) to verify edits

"edit_file" and "apply_patch" frequent failure cause on Windows repos: write_file formats newlines as '\n' (Unix LF). Windows files expect '\r\n' (CRLF). After write_file, the file has mixed/unexpected line endings. Subsequent edit_file searches fail because it reads raw bytes (which now contain bare \n), while the search string may have been constructed with \r\n expectations. Prevention: never use write_file on Windows repos — always use edit_file or apply_patch instead.

# Public Release Publishing Process

## One-time setup
- Public repo remote: `public` → `lordwedggie/xcp_derpNodes-release`
- Private repo remote: `github` → `lordwedggie/xcp_derpNodes`

## Steps for each release

### 1. Create release branch from private daily-development
```
git checkout daily-development
git pull github daily-development
git checkout -b release/vX.Y.Z
```

### 2. Remove private-only files
```
git rm -r .deepseek python/derpThemeManagerV2.py js/motha/themeManagerV2.js js/motha/themeManagerV2_core.js
```
(also `.monkeycode` if present)

### 3. Fix __init__.py
- Remove `from .python.derpThemeManagerV2 import ...`
- Remove `NODE_CLASS_MAPPINGS.update(THEME_V2_NODES)`
- Remove `**THEME_V2_DISPLAY` from display mappings
- Keep `bundled_asset_sync` import and `sync_bundled_assets()` call

### 4. Bump versions
Update `package.json` and `pyproject.toml` to new version

### 5. Commit and push
```
git add -A
git commit -m "release X.Y.Z"
git push public release/vX.Y.Z
```

### 6. Update public main
Since private and public main have unrelated histories:
```
git checkout main
git reset --hard release/vX.Y.Z
git push public main --force
```

### 7. Restore workflow file if needed
If force-push overwrote `.github/workflows/publish_action.yml`:
```
git checkout <old-main-commit> -- .github/workflows/publish_action.yml
git commit --amend --no-edit
git push public main --force
```

### 8. Trigger publish
Run `publish_action.yml` workflow_dispatch from `main` branch on GitHub Actions.

### Exclusions (never in public)
- `.deepseek/`
- `.monkeycode/`
- `python/derpThemeManagerV2.py`
- `js/motha/themeManagerV2.js`
- `js/motha/themeManagerV2_core.js`
- Never mention derpThemeManagerV2 removal in release notes or README
