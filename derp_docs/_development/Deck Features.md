Currently when we deck stacks to Decks, horizontal stacks decked to the top and bottom of the deck will match width with the deck, and when vertical stacks are decked to the deck's sides, they will match height with the total deck height including the heights from decked horizontal stacks, and thus a perfect rectangular shape is achieved. Our current handling of the deck is "Vertical Sandwich": Vertical Stacks on the sides are matching total height of deck height plus horizontal heights.

Implement "Horizontal Sandwich", where regardless of decking order (which stack is decked first, like how it is right now), vertical stacks will match height with the deck's height only, while horizontal stacks will match the total width of the deck's width, plus the width of decked vertical stacks.
3. "Automatic" will decide the deck is a vertical or horizontal sandwich automatically by the first stack that is being decked: if a horizontal stack is decked first to an empty deck, the deck becomes a vertical sandwich, if a vertical stack is decked first, the deck becomes a horizontal sandwich.
